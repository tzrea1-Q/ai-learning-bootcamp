# Text API 文档索引

这组文档用于补足项目根 `README.md` 没有展开写透的内容。

项目入口分工如下：

- `../README.md`
  - 项目入口、当前范围、关键跳转
- `../ENVIRONMENT.md`
  - 环境变量、启动方式、`.env` 模板和验证入口
- `05-已知问题与后续计划.md`
  - 当前限制、明确决策和下一步优先级

## 建议阅读顺序

1. `01-项目概览.md`
2. `02-架构与调用链.md`
3. `03-接口契约.md`
4. `04-开发测试调试指南.md`
5. `05-已知问题与后续计划.md`
6. `traces/README.md`

## 目录约定

- `docs/` 根目录：放项目本体文档、契约、调试指南、已知问题、索引
- `docs/traces/`：放测试、联调、回归、样例验证等留痕类文档和原始附件

## 文档说明

- `01-项目概览.md`
  - 说明这个项目是什么、当前能力边界、目录结构如何组织，以及各文档分别负责什么
- `02-架构与调用链.md`
  - 说明 FastAPI 路由、请求模型、Prompt 构造、OpenAI 兼容客户端、日志与响应清洗之间怎么衔接
- `03-接口契约.md`
  - 说明当前对外接口的 HTTP 方法、路径、请求体、响应体、错误语义和上游接口格式
- `04-开发测试调试指南.md`
  - 说明开发流程、运行测试、验证真实 API、阅读日志，以及做接口级调试
- `05-已知问题与后续计划.md`
  - 说明项目当前限制、已经确认的问题、需要优先推进的下一步工作
- `traces/README.md`
  - 说明各条测试、联调、回归和环境验证记录分别在验证什么

如果你只想快速上手，先看根目录 `README.md`。
如果你要继续开发或接手维护，先看 `../ENVIRONMENT.md`，再按本文档索引继续往下读。
