# Text API 文档索引

这组文档用于补足项目根 `README.md` 没有展开写透的内容。

## 文档治理约定

为了避免多处重复维护，当前文档分工固定如下：

- 项目根 `README.md`
  - 只做项目入口和导航，不再展开写完整环境配置或接口手册
- `../ENVIRONMENT.md`
  - 环境变量、`.env` 模板、启动方式、验证入口的唯一真相源
- `03-接口契约.md`
  - 接口、错误结构、输入边界、输出清洗规则的唯一真相源
- `04-开发测试调试指南.md`
  - 开发流程、测试、联调、日志和排查动作的唯一真相源
- `05-已知问题与后续计划.md`
  - 当前限制、明确决策和下一步优先级的唯一真相源

建议阅读顺序：

1. `01-项目概览.md`
2. `02-架构与调用链.md`
3. `03-接口契约.md`
4. `04-开发测试调试指南.md`
5. `05-已知问题与后续计划.md`

## 目录约定

- `docs/` 根目录：放项目本体文档、契约、调试指南、已知问题、索引
- `docs/traces/`：放测试、联调、回归、样例验证等留痕类文档和原始附件

- `01-项目概览.md`

  - 说明开发流程、运行测试、验证真实 API、阅读日志，以及做接口级调试
