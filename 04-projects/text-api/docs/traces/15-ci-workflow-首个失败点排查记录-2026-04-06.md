# CI workflow 首个失败点排查记录｜2026-04-06

## 1. 背景

本次检查对应 2026-04-06 的可选任务：

- 检查 `.github/workflows/text-api-pytest.yml` 的路径和依赖假设
- 提前排掉 1 个最可能导致 CI 首次失败的问题

## 2. 本次检查结果

工作流的目录假设本身没有问题：

- `working-directory: 04-projects/text-api`
- `requirements.txt`、`pytest.ini`、`tests/` 都在这个目录下

最可能的首次失败点，不在路径，而在 Python 版本假设：

- 原工作流使用 `python-version: "3.14"`
- 当前项目代码本身不依赖 3.14 特性
- 但 GitHub Actions 首次跑 CI 时，过新的 Python 版本更容易遇到 runner 可用性或三方依赖 wheel 支持波动

## 3. 本次修正

已做两处保守修正：

1. `python-version` 从 `3.14` 调整为 `3.12`
2. `pytest` 调用从裸命令改为 `python -m pytest`

同时补了 pip cache：

- `cache: pip`
- `cache-dependency-path: 04-projects/text-api/requirements.txt`

## 4. 这样改的理由

- `3.12` 对当前项目已经足够
- 比起继续追最新解释器版本，当前阶段更重要的是先把最小 CI 稳定跑通
- `python -m pytest` 能明确绑定到 Actions 当前设置好的 Python 解释器，减少 PATH 误判

## 5. 当前结论

- 路径假设：当前没有明显问题
- 依赖安装入口：当前没有明显问题
- 最可能的首个失败点：过新的 Python 版本假设
- 已提前做保守修正，优先保证 CI 首次通过概率
