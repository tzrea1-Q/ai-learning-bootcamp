# 接口契约

## 1. 对外接口风格

当前项目对外暴露的是标准的 `HTTP + JSON` REST 风格接口。

接口由 FastAPI 提供，调用方式是：

- `GET /health`
- `POST /summarize`
- `POST /key-points`
- `POST /rewrite`

## 2. 上游模型接口风格

项目调用 MiniMax 时，使用的是兼容 `OpenAI chat/completions` 形态的请求格式，而不是 OpenAI Responses API 格式。

核心结构如下：

```json
{
  "model": "MiniMax-M2.7",
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."}
  ]
}
```

这意味着当前项目更接近“OpenAI Chat Completions 兼容风格”的上游调用方式。

## 3. 通用请求体

三个文本处理接口当前都接收同样的请求结构：

```json
{
  "text": "需要处理的原始文本"
}
```

约束：

- `text` 必填
- `text` 不能是空字符串
- 纯空格会先 `strip()`，再按空输入处理

空输入当前会返回 `422 Unprocessable Entity`。

## 4. 通用响应体

三个文本处理接口当前都返回统一结构：

```json
{
  "task": "summarize | key-points | rewrite",
  "result": "模型处理后的正文",
  "model": "MiniMax-M2.7"
}
```

字段说明：

- `task`：当前执行的任务类型
- `result`：清洗后的最终输出
- `model`：使用的模型名

## 5. 各接口契约

### 5.1 `GET /health`

用途：

- 最小健康检查
- 验证服务是否已启动

响应：

```json
{
  "status": "ok"
}
```

### 5.2 `POST /summarize`

用途：

- 把一段原文压缩成 2 到 3 句话总结

示例请求：

```json
{
  "text": "FastAPI 是一个现代、高性能的 Python Web 框架，适合快速构建 API。"
}
```

示例响应：

```json
{
  "task": "summarize",
  "result": "FastAPI 是一个现代、高性能的 Python Web 框架，可快速构建 API，并自动生成交互式文档。",
  "model": "MiniMax-M2.7"
}
```

### 5.3 `POST /key-points`

用途：

- 从原文中提取 3 到 5 个关键要点

示例请求：

```json
{
  "text": "FastAPI 支持类型提示、自动文档和高性能，适合快速搭建 Web API。"
}
```

示例响应：

```json
{
  "task": "key-points",
  "result": "- 支持 Python 类型提示\n- 自动生成 API 文档\n- 具备高性能\n- 适合快速搭建 Web API",
  "model": "MiniMax-M2.7"
}
```

### 5.4 `POST /rewrite`

用途：

- 在不改变原意的前提下改写原文

示例请求：

```json
{
  "text": "FastAPI 可以更快做 API。"
}
```

示例响应：

```json
{
  "task": "rewrite",
  "result": "FastAPI 可以帮助开发者更高效地构建清晰、可靠的 API 服务。",
  "model": "MiniMax-M2.7"
}
```

## 6. 当前错误语义

当前项目还没有统一错误响应格式，但已经有基本错误语义：

### `422`

触发条件：

- 输入为空
- 请求体不合法

来源：

- FastAPI / Pydantic 校验层

### `500`

触发条件：

- 本地配置错误，例如 `MINIMAX_API_KEY` 缺失
- `MINIMAX_BASE_URL` 未设置

来源：

- 本地服务侧问题

### `502`

触发条件：

- 上游 HTTP 请求失败
- 上游返回结构不符合预期
- 去掉 `<think>` 后没有有效正文

来源：

- 模型供应商或上游通信问题

## 7. 当前输出清洗规则

在 `result` 返回给客户端之前，会执行以下处理：

1. 从 `choices[0].message.content` 中提取正文
2. 用正则去除 `<think>...</think>` 片段
3. 对结果做 `strip()`
4. 如果清洗后为空，则返回 `502`

这个清洗规则是当前接口契约的一部分，而不是临时 patch。

## 8. 当前契约的局限

虽然当前契约已经可用，但还存在这些限制：

- 没有 `code`、`message`、`detail` 这种统一错误结构
- 没有请求 id
- 没有版本号
- `rewrite` 输出风格可能偏扩写，不够稳定
- 还没有明确最大输入长度

这些问题不影响第 1 周最小闭环，但会影响后续扩展和对外稳定性。
