# Project Structure Standard

Each formal project should look roughly like this:

```text
project-name/
├─ app/
├─ tests/
├─ docs/
├─ scripts/
├─ .env.example
├─ Dockerfile
├─ docker-compose.yml
├─ requirements.txt
└─ README.md
```

## Minimum Requirements

- a `README.md`
- an `.env.example`
- a `tests/` directory
- local run instructions
- Docker support for formal projects when possible
