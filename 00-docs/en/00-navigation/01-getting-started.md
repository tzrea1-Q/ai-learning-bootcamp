# Getting Started

If today is your first day in this repository, do not browse randomly. Follow this order.

## Step 1: Confirm the Goal

Read:

- `00-docs/en/00-navigation/00-what-this-project-is.md`
- `00-docs/en/01-training-track/00-learning-overview.md`

You need to accept three facts:

- this is not a light AI overview
- this is an engineering-focused training system
- you are expected to build, document, and review your work continuously

## Step 2: Complete the Preflight Checklist

Read:

- `00-docs/en/01-training-track/01-preflight-checklist.md`
- `00-docs/en/01-training-track/02-tech-stack-and-environment.md`

Lock down your:

- accounts
- tools
- budget
- tech stack
- time blocks

## Step 3: Set Up the Environment

Follow:

- `00-docs/en/01-training-track/07-environment-setup-guide.md`

At minimum, you should get the following working:

- Git
- Python
- WSL2
- Docker
- FastAPI hello world
- PostgreSQL in Docker
- model API access

## Step 4: Build the Execution System

Read:

- `00-docs/en/02-execution-system/00-execution-system-overview.md`
- `00-docs/en/02-execution-system/daily-report-system/08-how-to-use-daily-reports.md`

Then start with:

- `01-templates/en/daily/daily-report-template.md`
- `01-templates/en/daily/daily-prompt-inputs.md`

## Step 5: Start the First Project

Read:

- `00-docs/en/01-training-track/03-12-week-plan.md`
- `00-docs/en/04-project-system/00-project-map.md`

Then start from `04-projects/week01-text-api` or your own week-one project.

## Step 6: Start Publishing Output

After your first week, begin using:

- `01-templates/en/weekly/engineering-weekly-report-template.md`
- `01-templates/en/blog/technical-blog-post-template.md`
- `01-templates/en/project/solution-document-template.md`
- `01-templates/en/postmortem/postmortem-template.md`
