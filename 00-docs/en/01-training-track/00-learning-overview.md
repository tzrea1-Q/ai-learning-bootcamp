# Learning Overview

## What This System Is Really Trying to Achieve

This is not an "AI roadmap" in the casual sense. It is a practical training system for beginners who want to become capable AI application builders.

After 12 weeks, you may not be a senior expert, but if you follow the system seriously, you should be able to:

- set up a complete AI development environment
- plan, build, test, and review your work consistently
- deliver small to mid-sized AI applications instead of only calling models
- turn your learning process into reports, blog posts, solution docs, and case studies
- use AI tools as part of engineering work without depending on them blindly

## The Five Capability Tracks

1. environment setup
2. execution discipline
3. project delivery
4. written output
5. engineering quality

## The Three Phases

### Weeks 1-4

- unify the stack
- build the first API delivery loop
- form habits around Git, README, testing, and Docker
- ship the first small usable AI project

### Weeks 5-8

- build and evaluate RAG systems
- understand failures with evidence
- gain database, state, and logging awareness
- learn controlled AI-assisted coding

### Weeks 9-12

- build workflows before agents
- add tool use
- turn projects into portfolio-ready assets

## Core Principles

- project work matters more than tutorials
- every week must leave evidence
- AI can assist but must not replace your judgment
- stability matters more than chasing advanced patterns too early
