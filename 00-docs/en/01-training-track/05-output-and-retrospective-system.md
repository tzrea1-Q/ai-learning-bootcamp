# Output and Retrospective System

The training system does not end at code. It requires a stable output loop:

`daily report -> weekly report -> blog draft -> solution document -> case study`

## Where Each Output Lives

- daily reports: `02-daily-reports/`
- weekly reports: `03-weekly-reports/`
- blog posts: `05-blog-posts/`
- solution docs: `06-solution-docs/`
- postmortems: `07-postmortems/`
- assets: `08-assets/`

## Minimum Weekly Output

- around five daily reports
- one weekly report
- one README update
- one reusable blog angle or draft
- one problem log or postmortem
