# Weekly Deliverables and Acceptance Criteria

## Minimum Weekly Deliverables

### Code

- real functional progress
- clear commit history
- runnable project state

### Documents

- README updated at least once
- one weekly report
- at least one output artifact such as a blog draft, solution doc, or postmortem

### Tests

- basic tests for core logic
- at least one validation pass for core interfaces

### Demo

- a short demo video or a reachable deployment

### Review

- a summary of progress, problems, decisions, and next steps

## Pass Line

A week passes when most of these are true:

- the project runs
- at least one core weekly goal is completed
- there is a README
- there are tests
- there are commits
- there is a demo
- there is a weekly report
