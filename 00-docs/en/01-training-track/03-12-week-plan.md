# 12-Week Plan

Each week must leave four kinds of evidence:

1. code or functional progress
2. document updates
3. tests or validation records
4. retrospectives or public output material

## Week 1

Build the smallest LLM API service.

## Week 2

Make the week-one project more reliable with logging, timeouts, error handling, and CI.

## Week 3

Build the first usable small product.

## Week 4

Introduce prompt comparison, format control, and basic evaluation.

## Weeks 5-6

Build and improve the first RAG system.

## Weeks 7-8

Add persistence, observability, and controlled AI-assisted feature development.

## Weeks 9-10

Build a workflow and tool-use system.

## Weeks 11-12

Build a minimal agent and package the final outputs into portfolio assets.
