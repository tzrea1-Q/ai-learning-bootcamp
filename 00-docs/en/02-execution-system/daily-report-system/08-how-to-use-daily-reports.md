# How to Use Daily Reports

## The Problem It Solves

Many learners:

- make plans that do not drive execution
- forget what they actually did
- restart from zero the next day
- fail to turn daily work into reusable output

The daily report system fixes that with a simple loop:

`plan in the morning -> execute during the day -> complete the report at night -> prepare tomorrow`

## Daily Flow

### Morning

1. create a new file in `02-daily-reports/`
2. copy `01-templates/en/daily/daily-report-template.md`
3. write today's plan, acceptance criteria, and risks
4. use the AI prompt inputs if needed

### Daytime

- do the Must tasks first
- record blockers when you are stuck
- update the report if the plan changes

### Night

Add:

- what was completed
- what was produced
- what problems occurred
- what you learned
- what tomorrow should start with
