# Weekly Rhythm

## Monday

- define the weekly goal
- break work into tasks
- confirm scope

## Tuesday to Thursday

- build core features
- integrate models and interfaces
- write tests
- fix issues

## Friday

- close engineering gaps
- update README
- clean config
- verify Docker and deployment readiness

## Saturday

- run tests
- review failures
- write retrospectives

## Sunday

- write the weekly report
- draft a blog post
- update the solution doc
- set the next minimum deliverable
