# Execution System Overview

Most people do not fail because they lack plans. They fail because they lack a loop.

The standard loop here is:

`plan -> execute -> record -> review -> extract -> plan again`

## Daily Loop

Every day:

1. write the plan first
2. execute by priority
3. complete the daily report at the end of the day
4. leave at least one reusable insight or artifact

## Weekly Loop

Every week:

1. summarize daily reports
2. write a weekly report
3. update a README or solution document
4. turn one problem or decision into a blog draft
5. write a postmortem or risk note
