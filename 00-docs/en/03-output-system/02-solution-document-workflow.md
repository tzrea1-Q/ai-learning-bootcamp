# Solution Document Workflow

Solution documents are not only for large teams. For beginners, they are useful because they force clearer thinking before implementation.

## When to Write One

- before starting a new project
- before building a large feature
- before integrating RAG, tool use, or workflow logic
- before changing multiple modules at once

## Minimum Sections

- background and goals
- users and scenarios
- input and output
- technical approach
- risks and boundaries
- minimum deliverable
