# 07｜详细可执行的环境部署指导

## 一、目标环境

你要搭建的是这样一套开发环境：

- **Windows 本地**：主开发桌面和 IDE
- **WSL2 Ubuntu**：Linux 一致性命令环境
- **Docker Desktop**：容器运行 PostgreSQL 和未来服务
- **Python + venv**：本地虚拟环境
- **Codex**：代码协作助手
- **MiniMax-M2.7**：主力模型 API
- **GitHub**：代码托管和 CI

这份文档的目标是让你在本地完成一个最小可用基准项目：

> FastAPI + PostgreSQL + Docker + MiniMax API + GitHub

---

## 二、推荐顺序

请严格按这个顺序安装，不要跳：

1. 安装 Windows 基础工具
2. 启用 WSL2 并安装 Ubuntu
3. 安装 Docker Desktop
4. 安装 Python 和 Git
5. 安装 IDE 并确认 Codex 可用
6. 配置 MiniMax API Key
7. 建立本地学习目录
8. 初始化第一个项目
9. 用 Docker 启 PostgreSQL
10. 起一个 FastAPI hello world
11. 测试 MiniMax API 调用
12. 推送到 GitHub

## 三、Windows 本地准备

### 1. 建议目录

先在 Windows 创建主目录，例如：

```powershell
mkdir E:\ai-learning-bootcamp
mkdir E:\ai-learning-bootcamp\00-docs
mkdir E:\ai-learning-bootcamp\01-templates
mkdir E:\ai-learning-bootcamp\02-weekly-reports
mkdir E:\ai-learning-bootcamp\03-postmortems
mkdir E:\ai-learning-bootcamp\04-projects
mkdir E:\ai-learning-bootcamp\05-assets
mkdir E:\ai-learning-bootcamp\06-archive
```

### 2. 安装 Git

安装后打开 PowerShell 检查：

```powershell
git --version
```

然后配置身份：

```powershell
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### 3. 安装 Python

安装完成后检查：

```powershell
python --version
pip --version
```

建议使用一个稳定主版本，不要在首月频繁切换版本。

### 4. 安装 Windows Terminal（如未安装）

确认你能方便切换：
- PowerShell
- Command Prompt
- Ubuntu (WSL)

## 四、启用 WSL2 与 Ubuntu

### 1. 以管理员身份打开 PowerShell

执行：

```powershell
wsl --install
```

如果系统提示需要重启，就重启。

### 2. 检查 WSL 状态

```powershell
wsl --status
wsl -l -v
```

你应该看到默认版本是 2，且 Ubuntu 已安装。

### 3. 首次进入 Ubuntu

在终端中打开 Ubuntu，完成 Linux 用户名和密码设置。

### 4. 在 Ubuntu 中更新系统

```bash
sudo apt update
sudo apt upgrade -y
```

### 5. 安装基础开发工具

```bash
sudo apt install -y python3 python3-pip python3-venv git curl build-essential make
```

### 6. 检查版本

```bash
python3 --version
pip3 --version
git --version
```

## 五、安装 Docker Desktop

### 1. 安装后检查 Windows 端

```powershell
docker --version
docker compose version
```

### 2. 确认 WSL 集成

打开 Docker Desktop 设置，确保：
- 启用了 WSL2 backend

### 3. 在 Ubuntu 中检查 Docker CLI

```bash
docker --version
docker compose version
```

如果 Ubuntu 中也能直接调用 Docker，说明联动正常。

### 4. 测试 Docker

```bash

```

## 十一、用 Docker 启 PostgreSQL

### 1. 创建 `docker-compose.yml`

```yaml
services:
  postgres:
    image: postgres:16
    container_name: ai_bootcamp_postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORE: postgres
      POSTGRES_DB: ai_bootcamp
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

### 2. 启动数据库

```bash
docker compose up -d
```

### 3. 检查容器

```bash
docker ps
```

### 4. 进入数据库验证

```bash
docker exec -it ai_bootcamp_postgres psql -U postgres -d ai_bootcamp
```

进入后可执行：

```sql
\dt
```

### 5. 退出

```sql
\q
```

## 十四、接入 MiniMax 的最小调用封装

你可以先写一个最小客户端模块，例如：

### `app/minimax_client.py`

```python
import os
from dotenv import load_dotenv
import requests

load_dotenv()

API_KEY = os.getenv("MINIMAX_API_KEY")
BASE_URL = os.getenv("MINIMAX_BASE_URL")


def minimax_chat(payloaE: dict) -> dict:
    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json",
    }
    response = requests.post(BASE_URL, headers=headers, json=payload, timeout=60)
    response.raise_for_status()
    return response.json()
```

> 这里的请求体结构与实际接口字段，请以你 MiniMax 控制台拿到的当前 API 文档为准。你的目标是先把“环境变量读取 + HTTP 请求 + 错误抛出”这条链路走通。

### 最小验证脚本 `scripts/test_minimax.py`

```python
from app.minimax_client import minimax_chat

payload = {
    "model": "MiniMax-M2.7",
    "messages": [
        {"role": "system", "content": "你是一个简洁的中文助手。"},
        {"role": "user", "content": "请用一句话解释什么是 FastAPI。"},
    ],
}

result = minimax_chat(payload)
print(result)
```

运行：

```bash
python scripts/test_minimax.py
```

如果返回成功，说明你的主力模型接入链路已通。

## 十七、最常见问题与处理建议

### 1. WSL 和 Windows 路径混乱
建议尽量固定一种主工作方式。最稳的是：
- 代码主要放 WSL 的 Linux 用户目录
- IDE 通过 WSL 打开项目
- Windows 只作为桌面和工具宿主

### 2. Docker 在 Windows 可用，WSL 里不可用
通常是 Docker Desktop 没开 WSL integration。去设置里打开 Ubuntu 集成。

### 3. `python` 命令找不到
在 Ubuntu 中优先使用 `python3`，并确认已安装 `python3-venv`。

### 4. PostgreSQL 5432 端口冲突
先用以下命令看占用：

```bash
docker ps
```

如果是已有本地数据库占用，就换映射端口，例如 `5433:5432`。

### 5. API Key 明明填了但请求失败
按顺序排查：
- `.env` 是否真的被加载
- Base URL 是否正确
- Header 是否正确
- 请求体字段是否与当前官方文档一致
- 是否有余额、权限或模型名错误
