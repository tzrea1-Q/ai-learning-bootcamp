# 06｜建议的本地学习目录结构

## 一、学习根目录建议

建议你在本地统一建立一个长期目录，例如：

```text
D:i-learning-bootcamp```

或者如果你希望代码主要放在 WSL 对应目录，也可以建立一个映射清晰的结构，例如：

```text
D:i-learning-bootcamp  00-docs  01-templates  02-weekly-reports  03-postmortems  04-projects  05-assets  06-archive```

## 二、推荐目录说明

### `00-docs`
放当前这套训练资料和你后续整理出的技术笔记：
- 学习总览
- 开工前清单
- 技术选型
- 周计划
- 架构草图
- 读书笔记

### `01-templates`
放所有可复用模板：
- README 模板
- 周报模板
- Postmortem 模板
- Case Study 模板
- `.env.example` 模板
- `docker-compose.yml` 模板

### `02-weekly-reports`
按周归档：
- week-01-report.md
- week-02-report.md
- ...

### `03-postmortems`
放失败案例：
- 端口冲突
- Docker 启动失败
- API 超时
- RAG 命中错误
- 结构化输出异常

### `04-projects`
放实际项目，建议每周或每阶段一个独立目录：
- `week01-text-api`
- `week03-meeting-assistant`
- `week05-rag-qa`
- `week09-workflow-assistant`
- `week11-mini-agent`

### `05-assets`
放图片、录屏、演示素材、流程图、接口截图：
- `demo-videos`
- `screenshots`
- `diagrams`

### `06-archive`
放旧版本、废弃尝试、过期方案，不要污染当前主线。

## 三、项目内推荐结构

每个正式项目目录建议长这样：

```text
project-name/
  app/
  tests/
  docs/
  scripts/
  .env.example
  Dockerfile
  docker-compose.yml
  pyproject.toml or requirements.txt
  README.md
```

### 目录职责
- `app/`：正式业务代码
- `tests/`：单元测试和集成测试
- `docs/`：架构、接口、实验记录
- `scripts/`：初始化、迁移、导入、辅助脚本

## 四、文件命名建议

### 周报
- `week-01-report.md`
- `week-02-report.md`

### 失败复盘
- `2026-04-xx-docker-network-issue.md`
- `2026-04-xx-rag-retrieval-failure.md`

### 项目案例
- `case-study-rag-qa.md`
- `case-study-mini-agent.md`

## 五、为什么要严格分层

如果你不把资料、模板、项目、复盘分开，几周后你会遇到：

- 不知道最新版本在哪
- 模板散落各处
- 项目和笔记混在一起
- 失败记录找不到
- 复用成本越来越高

你的目标不是临时做几个项目，而是建立一个 **可长期迭代的训练基地**。

## 六、建议的起步动作

你现在可以直接按下面顺序建目录：

1. 建立 `ai-learning-bootcamp` 根目录
2. 把本资料包全部放到 `00-docs`
3. 在 `01-templates` 中创建周报和 README 模板
4. 在 `04-projects` 中创建 `week01-text-api`
5. 从这个项目开始正式进入第 1 周训练
