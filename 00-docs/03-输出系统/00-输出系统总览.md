# 输出系统总览

## 一、为什么训练必须带输出系统

如果你只写代码，不整理文档，最终会出现两个问题：

- 你自己无法复盘成长路径
- 别人也看不到你的能力证据

所以这套训练要求你持续维护输出资产。

## 二、输出资产分为 5 类

1. 日报
2. 周报
3. 博客
4. 方案文档
5. 项目案例 / README / Postmortem

## 三、推荐产出链路

### 日报

记录事实和当天判断。

### 周报

整理一周的交付、问题和下一步。

### 博客

把一个具体问题、一个坑、一个判断写成可公开传播的文章。

### 方案文档

在项目开发前或中期，写清楚场景、方案、边界和风险。

### 项目案例

项目收尾时总结背景、方案、实现重点、结果与教训。

## 四、对应目录

- 日报：`02-daily-reports/`
- 周报：`03-weekly-reports/`
- 博客：`05-blog-posts/`
- 方案：`06-solution-docs/`
- 复盘：`07-postmortems/`

## 五、最低要求

每周至少留下一条能被未来复用的内容，不允许只留下“我学过”这种模糊记忆。
