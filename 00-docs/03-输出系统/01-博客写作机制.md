# 博客写作机制

## 一、博客不是额外任务，而是训练副产品

你不应该每周临时想"我要写什么博客"，而应该从日报、周报、项目文档和失败复盘里直接提取题材。

## 二、博客题材从哪里来

优先从下面这些素材里选：

- 一个具体 bug
- 一个模型调试对比
- 一个工程判断
- 一个失败案例
- 一个功能从 0 到 1 的实现过程
- 一轮工程化收口或阶段性取舍

## 三、好的博客应满足什么

- 有明确问题
- 有真实场景
- 有过程和判断
- 有结果和结论
- 有可复用价值

不要写成泛泛的概念摘要。
