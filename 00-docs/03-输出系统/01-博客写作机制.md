# 博客写作机制

## 一、博客不是额外任务，而是训练副产物

你不应该每周临时想“我要写什么博客”，而应该从日报和周报里直接提取题材。

## 二、博客题材从哪里来

优先从下面这些素材里选：

- 一个具体 bug
- 一个模型调试对比
- 一个工程判断
- 一个失败案例
- 一个功能从 0 到 1 的实现过程

## 三、推荐写作流程

1. 在日报里记下当天值得写的点
2. 周末从日报里挑 1 个最具体的问题
3. 用 `01-templates/blog/技术博文模板.md` 起草
4. 把截图、日志、代码片段放进 `08-assets/`
5. 博客初稿放到 `05-blog-posts/`

## 四、好的博客应该满足什么

- 有明确问题
- 有真实场景
- 有过程和判断
- 有结果和结论
- 有复用价值

不要写成泛泛的概念摘要。
