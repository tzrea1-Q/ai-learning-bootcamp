# 方案文档写作机制

## 一、为什么初学者也必须写方案文档

方案文档不是大公司专属。对初学者来说，写方案文档的价值更直接：

- 强迫你先想清楚再写代码
- 让项目边界更稳定
- 让 AI 协作更可控
- 给后续 README、Case Study 和复盘打底

## 二、什么时候写方案文档

以下场景建议先写方案：

- 开始一个新项目
- 做一个较大功能
- 接入 RAG / Tool Use / Workflow
- 改动影响多个模块

## 三、方案文档至少写清什么

- 背景和目标
- 用户与场景
- 输入输出
- 技术方案
- 数据与依赖
- 风险与边界
- 最小可交付

## 四、推荐流程

1. 在 `06-solution-docs/` 新建方案文档
2. 用 `01-templates/project/方案文档模板.md` 起草
3. 确认范围后再开始大规模开发
4. 开发过程中持续更新关键变更

## 五、最低要求

每个正式项目至少有 1 份方案文档，哪怕是轻量版。
