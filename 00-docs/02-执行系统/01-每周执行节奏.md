# 每周执行节奏

## 周一：定义本周目标

- 写本周最小可交付
- 拆分任务
- 确认本周项目边界
- 更新本周方案文档或任务列表

## 周二到周四：核心开发

- 实现功能
- 调接口和模型
- 写测试
- 修问题
- 更新关键文档

## 周五：工程化收口

- README
- `.env.example`
- 日志
- Docker
- 部署检查

## 周六：测试与复盘

- 跑测试
- 回看失败样例
- 整理问题清单
- 写复盘或 Postmortem

## 周日：整理公开输出

- 写周报
- 整理博客标题或初稿
- 更新方案文档
- 录 demo 或整理截图
- 制定下周最小可交付
