# 执行系统总览

## 一、这套执行系统解决什么问题

很多人的问题不是没有计划，而是没有闭环：

- 今天做什么不明确
- 做完没有记录
- 问题没有沉淀
- 周末说不清自己到底完成了什么

这套执行系统就是为了解决这个问题。

## 二、标准闭环

每天和每周都按下面链路走：

`计划 -> 执行 -> 记录 -> 复盘 -> 沉淀 -> 下一轮计划`

## 三、每日闭环

每天至少完成这 4 个动作：

1. 早上写日报里的今日计划
2. 白天按 Must / Should / Could 执行
3. 晚上补全实际完成、问题、收获和明日衔接
4. 留下至少一条可沉淀素材

## 四、每周闭环

每周至少完成这 5 个动作：

1. 汇总日报
2. 写周报
3. 更新项目 README 或方案文档
4. 整理博客草稿或标题池
5. 写失败复盘或风险记录

## 五、你需要长期维护的产出目录

- `02-daily-reports/`
- `03-weekly-reports/`
- `05-blog-posts/`
- `06-solution-docs/`
- `07-postmortems/`

## 六、最低执行版本

如果你当天很忙，也至少做到：

1. 写 3 条 Must 任务
2. 记录当天实际完成
3. 写明天第一件事做什么

只要不断链，训练系统就不会崩。
