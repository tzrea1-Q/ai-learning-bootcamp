# AI / LLM / Agent / Vibe Coding 高压训练资料包

这是一套面向 **12 周高压训练** 的本地学习资料。目标不是“学过一些 AI 概念”，而是把你训练成一个**能独立交付 AI 应用、具备中级工程习惯、可以持续迭代项目**的开发者。

## 资料目录

- `00-学习总览.md`：总目标、学习原则、执行方式
- `01-开工前准备清单.md`：开工前必须准备的账号、工具、环境、纪律
- `02-技术选型与环境方案.md`：Codex + MiniMax-M2.7 + Windows/WSL2 + Linux 部署方案
- `03-12周高压训练计划.md`：12 周主计划，按周拆解
- `04-每周交付与验收标准.md`：每周必须交付什么、如何判断是否通过
- `05-输出与复盘模板.md`：周报、README、失败复盘、项目案例模板
- `06-建议的本地学习目录结构.md`：你在本地创建学习目录时可直接参考的结构
- `07-详细可执行的环境部署指导.md`：Windows 本地 + WSL2 + Docker + Python + Codex + MiniMax 的详细落地步骤

## 推荐使用顺序

1. 先读 `00-学习总览.md`，建立训练框架和纪律。
2. 按 `01-开工前准备清单.md` 逐项勾选准备事项。
3. 用 `02-技术选型与环境方案.md` 锁定你的主力工具栈。
4. 按 `07-详细可执行的环境部署指导.md` 完成环境部署。
5. 完成后再进入 `03-12周高压训练计划.md` 的第 1 周执行。
6. 每周用 `04` 和 `05` 做验收与复盘。
7. 本地目录结构按 `06` 建立并长期沿用。

## 这套资料的核心原则

- **项目优先**：优先交付项目，不优先堆概念。
- **产出优先**：每周都必须留下代码、文档、测试、复盘、演示证据。
- **工程优先**：不接受只会调 API 的“玩具式 Demo”。
- **稳定优先**：能跑、能测、能部署、能解释，胜过功能很多但不稳定。
- **可复用优先**：所有目录、模板、环境都要服务于 12 周连续训练。

## 你现在最应该做的事情

- 创建本地学习根目录。
- 准备 GitHub、MiniMax API、Codex 使用环境。
- 配置 Windows + WSL2 + Docker 开发链路。
- 完成一个 FastAPI hello world + PostgreSQL + Docker 的最小闭环。
- 从第 1 周正式进入执行状态。
