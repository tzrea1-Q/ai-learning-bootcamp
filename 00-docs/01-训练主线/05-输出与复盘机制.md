# 05｜输出与复盘机制

## 一、为什么要单独建立输出系统

很多人做项目时只关注“今天写了什么代码”，结果 4 周后回头看：

- 做过什么说不清
- 哪些坑踩过找不到
- 想写博客时没有素材
- 想做作品集时没有整理过的文档

所以这套训练必须把“输出”当成主线，而不是附属品。

## 二、这套训练的输出链路

你每天和每周的输出应该按下面链路流动：

`日报 -> 周报 -> 博客草稿 -> 方案文档 -> Case Study / 作品集`

这条链路的意义是：

- 日报记录事实
- 周报整理判断
- 博客沉淀经验
- 方案文档训练结构化表达
- 案例文档服务对外展示

## 三、每类输出放在哪里

- 日报：`02-daily-reports/`
- 周报：`03-weekly-reports/`
- 博客：`05-blog-posts/`
- 方案文档：`06-solution-docs/`
- 失败复盘：`07-postmortems/`
- 截图、录屏、图表：`08-assets/`

## 四、你每周至少要留下什么

### 最低版本

- 5 份左右日报
- 1 份周报
- 1 份 README 更新
- 1 条博客选题或半成品草稿
- 1 条问题记录或失败复盘

### 进阶版本

再额外补：

- 1 份方案文档
- 1 份项目案例
- 1 段 demo 视频

## 五、输出物如何相互复用

### 日报 -> 周报

从日报里提取：

- 本周完成项
- 本周阻塞
- 本周最重要的判断
- 下周衔接

### 日报 / 周报 -> 博客

从执行记录里提取：

- 某个具体问题
- 你如何定位
- 你做了什么取舍
- 最后得到什么结论

### 项目开发 -> 方案文档

在开发前或中途整理：

- 场景
- 输入输出
- 技术方案
- 风险
- 交付范围

### 项目结束 -> Case Study

项目做完后，把过程整理成：

- 背景
- 方案
- 实现重点
- 结果
- 问题
- 下一步

## 六、模板在哪里

统一在 `01-templates/`：

- `daily/`
- `weekly/`
- `blog/`
- `project/`
- `postmortem/`
- `case-study/`

## 七、最低执行要求

如果你很忙，也至少要做到：

1. 当天写日报
2. 周末写周报
3. 每周留下一条可公开表达的素材
4. 每个正式项目至少写 README 和方案文档

只要这条链不断，你的学习就会逐步变成作品积累。
