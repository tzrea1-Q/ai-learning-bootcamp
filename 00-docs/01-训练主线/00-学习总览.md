# 00｜学习总览

## 一、这套训练真正要达成什么

这不是一份“AI 概念入门材料”，而是一套面向初学者的 AI 开发训练系统。

12 周后，你不一定会变成资深专家，但如果严格执行，应该能把自己推进到这个状态：

- 能独立配置一套 AI 开发环境
- 能围绕一个目标持续做计划、开发、测试、复盘
- 能交付中小型 AI 应用，而不是只会调模型聊天
- 能把学习过程沉淀成日报、周报、博客、方案和案例
- 能把 AI 工具纳入工程流程，而不是完全依赖它

一句话说清楚：

> 目标不是“学过 AI”，而是“具备 AI 应用开发与持续输出能力”。

## 二、这套训练会同时训练 5 条能力线

### 1. 环境能力

- 配置 Windows / WSL2 / Docker / Python / Git / API Key
- 跑通本地开发、测试和部署链路

### 2. 执行能力

- 每天先写计划
- 白天按优先级执行
- 晚上补全结果和问题

### 3. 项目能力

- 从最小 API 做到完整应用
- 再做到 RAG、Workflow、Tool Use、最小 Agent

### 4. 输出能力

- 写日报
- 写周报
- 写博客
- 写方案文档
- 写项目 README / Case Study / Postmortem

### 5. 工程能力

- 测试
- 日志
- Docker
- 环境变量管理
- 版本管理
- 失败复盘

## 三、12 周的阶段目标

### 第 1 阶段：第 1~4 周

重点是建立基础工程能力：

- 统一技术栈
- 打通 API 开发闭环
- 建立 Git、README、测试、Docker 习惯
- 完成第一个可运行、可演示的小项目

### 第 2 阶段：第 5~8 周

重点是形成 AI 应用中级工程手感：

- 理解并实现 RAG
- 做基本评测
- 能解释失败原因
- 形成数据库、状态、日志意识
- 学会可控地使用 AI 编码协作

### 第 3 阶段：第 9~12 周

重点是系统化与作品集化：

- 先做 Workflow，再做有限自主 Agent
- 接入工具调用
- 输出完整项目方案、README、复盘和案例
- 整理成对外可展示的作品集

## 四、最重要的学习原则

### 1. 项目比教程更重要

任何资料输入都必须服务于本周项目。

### 2. 每周必须留下证据

每周至少要留下这些证据中的多项：

- commit
- 可运行代码
- 测试结果
- README 更新
- 周报
- 博客草稿
- 方案文档
- 失败复盘

### 3. AI 只能辅助，不能替你思考

你可以大量使用 Codex、ChatGPT、Claude、Copilot 一类工具，但必须满足：

- 关键逻辑自己能解释
- 改动前后有版本记录
- 代码合并前自己读过
- 出错时自己先定位问题

### 4. 先稳定，再高级

不要一开始就追求“多 Agent”“全自动”。先做：

- 明确输入
- 明确输出
- 明确错误处理
- 明确测试边界
- 明确项目价值

## 五、推荐执行方式

### 每周投入

建议每周投入 `20~25` 小时，最低不低于 `15` 小时。

### 每周固定节奏

- 周一：确定目标和任务拆解
- 周二到周四：核心开发
- 周五：工程化收口
- 周六：测试、评测、复盘
- 周日：写周报、整理博客素材、制定下周计划

## 六、12 周结束时的理想成果

你至少应拿到：

- 2 个可运行、可演示的 AI 项目
- 1 个带 RAG 的项目
- 1 个多步骤 Workflow / Agent 系统
- 一套稳定模板与目录结构
- 一组日报 / 周报 / 博客 / 方案 / 复盘文档
- 一份可展示的作品集素材包

## 七、开始前的最低门槛

开始这套训练前，你至少需要：

- 会一种编程语言，建议 Python
- 对 Git、HTTP、数据库有基础认知
- 愿意接受严格验收，不靠“差不多”过关
- 愿意持续记录与复盘，而不是只写代码
