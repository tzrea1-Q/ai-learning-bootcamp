# 06｜建议的本地学习目录结构

## 一、推荐根目录

建议你长期维护一个固定训练根目录，例如：

```text
E:\ai-learning-bootcamp
```

或者在 WSL 中：

```text
~/ai-learning-bootcamp
```

重点不是放在哪，而是整套结构长期稳定。

## 二、推荐目录结构

```text
ai-learning-bootcamp/
├─ 00-docs/
├─ 01-templates/
├─ 02-daily-reports/
├─ 03-weekly-reports/
├─ 04-projects/
├─ 05-blog-posts/
├─ 06-solution-docs/
├─ 07-postmortems/
├─ 08-assets/
└─ 09-archive/
```

## 三、每个目录负责什么

### `00-docs`

放训练方法、执行规则、项目规范和导航文档。

### `01-templates`

放所有复用模板：

- 日报
- 周报
- 博客
- README
- 方案文档
- 失败复盘
- 项目案例

### `02-daily-reports`

每天 1 份日报，记录计划、执行、问题、收获和明日衔接。

### `03-weekly-reports`

每周 1 份周报，做验收与复盘。

### `04-projects`

放实际项目代码，建议按周或按阶段拆分。

### `05-blog-posts`

放博客草稿、标题池和可公开发布的文章。

### `06-solution-docs`

放方案文档、架构草案、需求拆解、设计说明。

### `07-postmortems`

放失败案例和重大问题复盘。

### `08-assets`

放录屏、截图、架构图、流程图、接口示意图。

### `09-archive`

放旧版本、废弃方案、阶段归档，不要污染主线。

## 四、项目内推荐结构

每个正式项目目录建议长这样：

```text
project-name/
├─ app/
├─ tests/
├─ docs/
├─ scripts/
├─ .env.example
├─ Dockerfile
├─ docker-compose.yml
├─ requirements.txt
└─ README.md
```

## 五、命名建议

### 日报

- `2026-04-02.md`
- `2026-04-03.md`

### 周报

- `week-01-report.md`
- `week-02-report.md`

### 博客

- `2026-04-fastapi-health-check.md`
- `2026-04-rag-chunking-notes.md`

### 方案文档

- `week03-简历助手-方案.md`
- `week05-rag-问答系统-方案.md`

### 失败复盘

- `2026-04-docker-network-issue.md`
- `2026-04-rag-retrieval-failure.md`

## 六、为什么一定要分层

不分层，几周后你会遇到：

- 模板找不到
- 日报和项目混在一起
- 想写博客时素材散落四处
- 失败记录无法复用
- 项目越来越难维护

你的目标不是临时做几个 demo，而是建立一个可长期迭代的训练基地。
