# 01｜开工前准备清单

## 一、总原则

开工前不需要把所有工具一次性买齐，但你必须准备好一套能稳定执行 12 周的最小组合。

你的目标不是“账号很多”，而是：

- 环境能跑
- API 能调
- 项目能做
- 文档能写
- 产出能沉淀

## 二、必须准备的账号

### 1. GitHub

用途：代码托管、版本管理、issue、Actions、作品集展示。

必须完成：

- 注册账号
- 开启 2FA
- 学会 `clone / commit / push / branch / PR`

### 2. 主力模型 API 账号

当前建议主力是 `MiniMax-M2.7`。

必须完成：

- 能进入控制台
- 创建 API Key
- 确认额度或计费方式
- 记录 Key 的安全存放位置

### 3. 备用模型 API 账号

不是首日必须，但建议在第 `4~6` 周前准备好至少一个备用模型源，用于：

- 效果对比
- 失败样例排查
- 模型行为验证

### 4. Codex / AI 编码协作账号

必须完成：

- 明确你如何在 IDE 中接入 Codex
- 测试一次简单读写或重构任务

### 5. 云部署账号

用途：后续部署 Linux 环境中的 API 或 Web 服务。

至少要有一个可用平台账号。

## 三、必须安装的工具

### Windows 本地

- Git
- Python
- Docker Desktop
- 支持 Codex 的 IDE
- WSL2
- Ubuntu（WSL 发行版）
- PowerShell 或 Windows Terminal

### WSL2 中建议安装

- Python 3
- pip
- venv
- git
- curl
- build-essential

## 四、必须建立的目录

开始前就把训练基地搭好：

- `00-docs/`
- `01-templates/`
- `02-daily-reports/`
- `03-weekly-reports/`
- `04-projects/`
- `05-blog-posts/`
- `06-solution-docs/`
- `07-postmortems/`
- `08-assets/`
- `09-archive/`

## 五、必须准备的模板

你至少要提前确认这些模板可用：

- 日报模板
- 周报模板
- 博客模板
- README 模板
- 方案文档模板
- 失败复盘模板
- 项目案例模板

统一位置：`01-templates/`

## 六、必须锁定的技术栈

开工前不要摇摆，先锁定主线：

- 语言：Python
- 后端：FastAPI
- 数据库：PostgreSQL
- ORM：SQLAlchemy
- 迁移：Alembic
- 测试：pytest
- 容器：Docker
- CI：GitHub Actions
- 代码助手：Codex
- 主力模型 API：MiniMax-M2.7
- 开发环境：Windows 本地 + WSL2 + Docker

## 七、必须建立的纪律

1. 每天必须有真实进展，而不是只看资料
2. 每周必须有 README、测试和总结
3. 所有密钥必须用环境变量管理
4. 每周必须沉淀至少一条可公开输出的素材
5. 每周必须做一次验收和复盘

## 八、开工前自检

- [ ] GitHub 已可正常使用
- [ ] 模型 API Key 已拿到
- [ ] Codex 在 IDE 中可用
- [ ] Windows + WSL2 + Docker 可正常联动
- [ ] Python 虚拟环境可创建
- [ ] PostgreSQL 可通过 Docker 启动
- [ ] FastAPI hello world 能跑
- [ ] 训练目录结构已建立
- [ ] 模板目录已建立
- [ ] 已为未来 12 周锁定固定时间块

## 九、你现在就要做的事

1. 把未准备项列成自己的 checklist
2. 进入 `07-详细可执行的环境部署指导.md`
3. 环境完成后，创建第一周项目和第一份日报
