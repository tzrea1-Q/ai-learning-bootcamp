# 训练主线

这里放的是整套训练计划的主干文档。

如果你只想知道“这 12 周到底怎么跑”，先看这个目录。

## 阅读顺序

1. `00-学习总览.md`
2. `01-开工前准备清单.md`
3. `02-技术选型与环境方案.md`
4. `07-详细可执行的环境部署指导.md`
5. `03-12周高压训练计划.md`
6. `04-每周交付与验收标准.md`
7. `05-输出与复盘机制.md`

## 每个文件解决什么问题

- `00-学习总览.md`：这套训练到底要把你带到什么程度
- `01-开工前准备清单.md`：开始前必须准备好的账号、工具、纪律
- `02-技术选型与环境方案.md`：统一技术栈，避免来回摇摆
- `03-12周高压训练计划.md`：按周拆分的训练路线
- `04-每周交付与验收标准.md`：每周怎样算通过
- `05-输出与复盘机制.md`：日报、周报、博客、方案、复盘如何形成闭环
- `07-详细可执行的环境部署指导.md`：从零把开发环境搭起来

## 你读完这里后应该能做到

- 明确训练目标和边界
- 搭起一致的开发环境
- 知道每周应该交付什么
- 知道每天和每周要留下哪些文档资产
