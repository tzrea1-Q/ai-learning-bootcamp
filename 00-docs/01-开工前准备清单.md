# 01｜开工前准备清单

## 一、总原则

开工前你不需要把所有付费订阅一次性买齐，但你必须准备好 **能连续 12 周执行** 的最小稳定组合。

你的目标不是“账号齐全”，而是：

- 环境能跑
- API 能调
- 代码能管
- 项目能测
- 应用能部署
- 节奏能持续

## 二、必须准备的账号

### 1. GitHub
用途：代码托管、issue 管理、Actions、作品集展示。

必须完成：
- 注册账号
- 开启 2FA
- 建立一个专用学习仓库或组织规则
- 学会 clone / commit / push / branch / PR 基本操作

### 2. MiniMax 开发者账号
用途：作为主力模型 API 提供商。

必须完成：
- 注册并能进入控制台
- 创建 API Key
- 确认计费方式或可用额度
- 记录 Key 存放位置

### 3. OpenAI / 备用模型提供商账号（建议）
用途：后期做对比实验、排查模型特性差异。

不是首日必须，但建议在第 4~6 周前准备好至少一个备用模型源。

### 4. Codex 可用账号 / IDE 登录方式
用途：代码协作、重构、补测试、文档生成。

必须完成：
- 明确你用哪种方式接入 Codex
- 确认 IDE 内可用
- 测试一次对当前仓库的简单读写建议

### 5. 云部署账号
用途：部署 Linux 环境中的 API 或 Web 服务。

必须完成：
- 至少一个可用平台账号
- 明确后续第 3 周开始可以部署项目

## 三、必须安装的工具

### Windows 本地必须有
- Git
- Python
- Docker Desktop
- 支持 Codex 的 IDE
- WSL2
- Ubuntu（WSL 发行版）
- 一个终端工具（PowerShell / Windows Terminal）

### WSL2 中建议安装
- Python 3
- pip
- venv
- git
- curl
- build-essential
- make（可选）

### 开发期强烈建议
- Postman 或 Hoppscotch（二选一）
- 一个密码管理器或安全的密钥保存方式
- 一个录屏工具
- 一个时间管理或任务管理工具

## 四、必须准备的本地目录与模板

你要在本地提前准备：

- 学习根目录
- docs 目录
- projects 目录
- templates 目录
- weekly-reports 目录
- postmortems 目录

你还要提前准备好这些模板：

- README 模板
- 周报模板
- 失败复盘模板
- 项目案例模板
- `.env.example` 模板
- Git commit 规范

## 五、必须锁定的技术栈

开工前不要摇摆，你应先锁定：

- 语言：Python
- 后端：FastAPI
- 数据库：PostgreSQL
- ORM：SQLAlchemy
- 迁移：Alembic
- 测试：pytest
- 容器：Docker
- CI：GitHub Actions
- 代码助手：Codex
- 主力模型 API：MiniMax-M2.7
- 开发环境：Windows 本地 + WSL2 + Docker
- 部署：Linux 云服务器或 Linux 托管平台

## 六、必须建立的纪律

### 1. 每天必须有 commit
不是改空格，而是有效进展。

### 2. 任何项目都必须有 README
没有 README，视为项目不完整。

### 3. 所有密钥都必须用环境变量管理
绝不允许把 Key 写死在代码里。

### 4. 每周都必须有测试结果
最少也要有基本单元测试和一次集成验证。

### 5. 正式项目必须部署
不允许一直停留在本地。

### 6. 每周必须有复盘
做完不复盘，等于没训练工程判断力。

## 七、预算准备

建议你提前准备一份简单预算表，记录：

- Codex 相关使用成本
- MiniMax API 成本
- 部署成本
- 域名成本（首月通常非必须）
- 额外工具成本

目标不是省到极限，而是知道每一周的花费和产出是否匹配。

## 八、开工前自检

在正式开始前，逐项确认：

- [ ] GitHub 已可正常使用
- [ ] MiniMax API Key 已拿到
- [ ] Codex 在 IDE 中可用
- [ ] Windows + WSL2 + Docker 可正常联动
- [ ] Python 虚拟环境可创建
- [ ] PostgreSQL 可通过 Docker 启动
- [ ] FastAPI hello world 能跑
- [ ] 本地目录结构已建立
- [ ] README / 周报 / Postmortem 模板已建立
- [ ] 你已经为未来 12 周锁定固定时间块

## 九、你现在就要做的事

1. 读完本文件后，把未准备项列成 checklist。
2. 直接进入 `07-详细可执行的环境部署指导.md`。
3. 完成环境后，创建第一个基准项目仓库。
