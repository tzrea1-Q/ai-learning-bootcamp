# 03｜12周高压训练计划

## 总原则

这 12 周不是“自由发挥式学习”，而是分阶段、按周压进度的训练计划。每周都必须有明确主题、交付物、工程要求和验收线。

---

## 第 1 周：重建开发者习惯，做最小 LLM API 服务

### 本周目标
- 统一环境
- 起一个 FastAPI 服务
- 接入 MiniMax-M2.7
- 完成 3 个基础文本处理接口
- 建立测试、README、Docker、Git 习惯

### 交付
- 一个 `text-api` 项目
- 至少 3 个接口：总结、改写、要点提取
- `README.md`
- `.env.example`
- 5 个以上测试
- Docker 启动方式

### 训练重点
- 请求 / 响应模型
- prompt 模板
- 基础异常处理
- Git 规范

---

## 第 2 周：把“能跑”提升到“像样”

### 本周目标
- 让第 1 周项目更可靠
- 补日志、重试、超时、错误码、CI

### 交付
- 统一响应格式
- 基础 logging
- 异常处理增强
- GitHub Actions 跑测试
- Docker 稳定化

### 训练重点
- 工程可靠性意识
- 基础自动化
- 对错误和失败的控制

---

## 第 3 周：做第一个完整小产品

### 本周目标
从“API 服务”提升到“用户可用应用”。

### 可选项目
- 简历优化助手
- 会议纪要整理器
- 客服回复草稿助手
- 长文总结工具

### 交付
- 前后端最小闭环
- 历史记录
- 数据持久化
- 可上线版本
- 1 段演示视频

### 训练重点
- 用户视角
- 链路完整性
- 项目边界定义

---

## 第 4 周：Prompt、格式约束与效果评测

### 本周目标
停止“凭感觉调模型”，开始做基础评测。

### 交付
- 20~30 条测试样例
- 至少 3 组 prompt 对比
- 至少 2 种输出格式策略对比
- 一份失败案例记录

### 训练重点
- structured output
- few-shot 思维
- 失败类型分类
- 有证据地改进

---

## 第 5 周：第一个 RAG 系统

### 本周目标
做一个最小可用的文档问答系统。

### 交付
- 文档导入
- 文档切块
- 检索
- 基于检索结果回答
- 引用片段展示

### 训练重点
- chunking
- retrieval
- 来源引用
- 无答案场景控制

---

## 第 6 周：RAG 优化与实验

### 本周目标
把 RAG 从“能用”提升到“知道为什么有用或没用”。

### 交付
- 至少 30 个问题的 RAG 测试集
- chunk size 对比
- top-k 对比
- 检索拼接策略对比
- 一份实验结论

### 训练重点
- 实验意识
- 错误定位
- 检索失败 vs 回答失败的区分

---

## 第 7 周：数据库、状态与可观测性

### 本周目标
让系统开始像“真实服务”而不是“临时脚本”。

### 交付
- PostgreSQL 正式接入
- 请求记录表
- 任务记录表
- 评测记录表
- 日志增强
- 基础管理接口或查询能力

### 训练重点
- 状态持久化
- 可追踪性
- 数据驱动优化

---

## 第 8 周：可控的 Vibe Coding / Agent 式编码协作

### 本周目标
训练你使用代码代理时的控制力。

### 建议新功能
- 历史搜索
- 导出报告
- 用户模板
- 批量处理
- 管理后台雏形

### 交付
- 一项由 Codex 重度参与但可控完成的新功能
- 测试补全
- 风险点记录

### 训练重点
- 需求先行
- 小步修改
- 自己审代码
- 自己保回滚点

---

## 第 9 周：先做 Workflow，不急着做 Agent

### 本周目标
做一个多步骤工作流系统。

### 可选方向
- 研究资料整理工作流
- 日报生成系统
- 会议材料提炼流程

### 交付
- 清晰的步骤链
- 每步输入输出说明
- 中间状态记录
- 可单步调试

### 训练重点
- 流程分层
- 可控性
- 步骤编排

---

## 第 10 周：Tool Use 与函数调用

### 本周目标
让系统不只是“说”，而是“能做”。

### 至少接入 2 类工具
- 查数据库
- 检索本地文档
- 保存结果
- 调用外部 API
- 导出文件

### 交付
- 工具调用链路
- 参数校验
- 失败处理
- 调用日志

### 训练重点
- 何时调用工具
- 何时不调用工具
- 错误恢复

---

## 第 11 周：最小 Agent 系统

### 本周目标
实现一个有限自主的 Agent。

### 可选方向
- 自动研究助理
- 邮件处理助手
- 任务拆解系统
- 文档理解与行动建议助手

### 交付
- 明确目标
- 工具列表
- 中间状态
- 决策路径
- 最终输出
- 失败路径处理

### 训练重点
- 有限自主
- Agent 与 Workflow 的边界
- 稳定性与成本权衡

---

## 第 12 周：收尾、整理、作品集化

### 本周目标
把训练成果变成可展示、可复盘、可继续迭代的工程资产。

### 交付
- 至少 2 个整理完善的项目
- 最终 README
- 架构图
- 测试说明
- 已知问题
- 技术复盘
- 项目 Case Study
- 失败总结
- 作品集页

### 训练重点
- 收尾能力
- 技术表达能力
- 作品集组织能力

---

## 全程要求

12 周中，每周都必须做到：

- 有 commit
- 有文档
- 有测试
- 有演示
- 有复盘
- 有最小可交付

## 12 周结束时你的理想状态

你不一定会成为真正意义上的“资深开发者”，但你应该具备这些能力：

- 能独立交付中小型 AI 应用
- 能解释自己的系统设计
- 能做基本评测和迭代
- 能在 Linux 环境上线
- 能写出像样的技术文档和项目案例
