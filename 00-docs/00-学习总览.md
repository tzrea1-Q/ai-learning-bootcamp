# 00｜学习总览

## 一、这 12 周的真正目标

这不是一份“AI 入门路线图”，而是一份 **高压工程训练计划**。你在 12 周后不可能自动变成“资深专家”，但如果你严格执行，可以把自己提升到下面这个位置：

- 能独立设计并实现一个中小型 AI 应用。
- 能从需求、接口、数据、模型、日志、测试、部署完整走通。
- 能区分 Prompt、RAG、Workflow、Tool Use、Agent 的适用边界。
- 能使用 AI 代码助手提高效率，但不被其控制。
- 能持续产出项目、文档、复盘和公开成果。

换句话说，这 12 周的目标不是“了解 AI”，而是：

> 从一个会使用 AI 工具的人，训练成一个能交付 AI 应用的开发者。

## 二、阶段性目标

### 第 1 阶段：第 1~4 周
重点是建立基础工程能力：

- 统一技术栈
- 打通 API 开发闭环
- 形成 Git、README、测试、日志、Docker 的基础习惯
- 完成第一个可上线的小型 LLM 应用

### 第 2 阶段：第 5~8 周
重点是开始具备 AI 应用中级工程手感：

- 理解并实现 RAG
- 学会基本评测
- 学会分析失败原因
- 建立数据库、状态、可观测性意识
- 学会可控地使用 Vibe Coding / 代码代理

### 第 3 阶段：第 9~12 周
重点是系统化与作品集化：

- 先做 Workflow，再做有限自主 Agent
- 接入工具调用
- 形成多步骤任务链路
- 梳理架构、测试、上线、文档与复盘
- 输出完整项目案例和作品集

## 三、学习原则

### 1. 项目比教程更重要
每周必须围绕项目产出学习。任何资料输入都必须服务于本周项目需求。

### 2. 每周必须有可展示结果
每周必须至少留下这些证据中的 3 项：

- GitHub commit
- 一段可运行代码
- 测试结果
- README 更新
- 一段演示视频
- 一篇周报
- 一份失败复盘

### 3. AI 只能辅助，不能替你思考
你可以大量使用 Codex、ChatGPT、Claude、Copilot 一类工具，但必须满足：

- 关键逻辑自己能解释
- 改动前后有 Git checkpoint
- 代码合并前自己读过
- 出错时自己先定位问题

### 4. 先稳定，再高级
不要追求“多 Agent”“全自动”“通用智能体”。先做：

- 明确输入
- 明确输出
- 明确错误处理
- 明确测试边界
- 明确项目价值

### 5. 训练的是工程能力，不是工具熟练度
你不是来学某个聊天产品怎么更好用，而是训练：

- API 设计
- 数据建模
- 模型接入
- RAG 设计
- Tool Use
- Workflow 组织
- 测试
- 部署
- 复盘

## 四、执行方式

### 每周投入要求
建议每周投入 **20~25 小时**。最低不低于 **15 小时**。低于这个强度，很难完成高压版本的目标。

### 每周固定节奏

#### 周一：设计日
- 写出本周目标
- 明确用户、场景、输入、输出、边界
- 拆成 issue / task list

#### 周二到周四：核心开发日
- 实现功能
- 写接口
- 调模型
- 写测试
- 处理 bug

#### 周五：工程化收口日
- README
- `.env.example`
- Docker
- 日志
- GitHub Actions
- 错误处理

#### 周六：评测与复盘日
- 跑测试
- 做效果对比
- 记录失败案例
- 整理日志

#### 周日：发布与计划日
- 更新项目文档
- 录演示视频
- 写周报
- 制定下周最小可交付

## 五、你的输出目标

12 周结束时，至少应拿到：

- 2 个可访问、可演示的 AI 应用
- 1 个带 RAG 的应用
- 1 个多步骤 Workflow / Agent 系统
- 一套统一工程模板
- 一组评测样例与实验记录
- 一套周报 / README / Postmortem / Case Study 文档
- 一份可展示的作品集

## 六、你必须持续问自己的四个问题

不要再问“今天学了多少概念”，你要每周问：

1. 这周我交付了什么？
2. 这周我修掉了什么？
3. 这周我验证了什么？
4. 这周我写清楚了什么？

只要你能持续回答这四个问题，你就会从“学习者心态”逐渐转向“开发者心态”。

## 七、开练前的最低门槛

开始这套训练前，你至少需要：

- 会一种编程语言，建议 Python
- 对 Git、HTTP、数据库有基础认知
- 愿意接受严格验收，不靠“差不多”过关
- 允许自己在 12 周里持续被 bug、失败样例、重构打脸
