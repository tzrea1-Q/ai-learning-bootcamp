# 新手启动路径

如果你今天第一次进入这个仓库，不要到处乱看。按下面顺序执行。

## 第一步：确认目标

先读：

- `00-docs/00-导航/00-项目定位与使用方式.md`
- `00-docs/01-训练主线/00-学习总览.md`

你要先确认自己接受这套训练的核心事实：

- 这不是“轻松学一点 AI”
- 这是一套偏工程化、偏交付的训练系统
- 你需要持续写代码、文档、复盘和公开输出

## 第二步：完成开工前准备

读：

- `00-docs/01-训练主线/01-开工前准备清单.md`
- `00-docs/01-训练主线/02-技术选型与环境方案.md`

你要在开始前锁定：

- 账号
- 工具
- 预算
- 技术栈
- 固定时间块

## 第三步：搭环境

直接按：

- `00-docs/01-训练主线/07-详细可执行的环境部署指导.md`

做完后，你至少要跑通：

- Git
- Python
- WSL2
- Docker
- FastAPI hello world
- PostgreSQL 容器
- 模型 API 调用

## 第四步：建立执行系统

接着读：

- `00-docs/02-执行系统/00-执行系统总览.md`
- `00-docs/02-执行系统/日报机制/08-日报机制使用说明.md`

然后复制模板：

- `01-templates/daily/日报模板.md`
- `01-templates/daily/日报输入模板.md`

你的目标是让自己从第一天开始，就有计划、有记录、有复盘。

## 第五步：进入第 1 周项目

然后看：

- `00-docs/01-训练主线/03-12周高压训练计划.md`
- `00-docs/04-项目系统/00-项目地图.md`

从 `04-projects/text-api` 或你自己的第 1 周项目开始。

## 第六步：开始沉淀输出

执行一周后，开始使用：

- `01-templates/weekly/工程周报模板.md`
- `01-templates/blog/技术博文模板.md`
- `01-templates/project/方案文档模板.md`
- `01-templates/postmortem/失败复盘模板.md`

## 你今天的最小动作

如果你现在时间不多，今天只做这 3 件事：

1. 看完 `00-学习总览.md`
2. 按环境部署文档跑通本地开发环境
3. 新建一份今天的日报
