# 仓库地图与使用顺序

说明：

- 这是一张补充地图，不是仓库总入口
- 第一次进入仓库，先看根目录 `README.md`
- 第一次真正开始执行，再看 `01-新手启动路径.md`

## 仓库的五条主线

### 1. 导航线
位置：`00-docs/00-导航/`

解决第一次使用仓库时的方向问题。

### 2. 训练线
位置：`00-docs/01-训练主线/`

解决 12 周训练目标、节奏、技术栈和验收问题。

### 3. 执行线
位置：`00-docs/02-执行系统/`

解决每天怎么做、每周怎么跑、如何形成闭环的问题。

### 4. 输出线
位置：`00-docs/03-输出系统/` 与 `01-templates/`

解决博客、方案、周报、复盘怎么写的问题。

### 5. 项目线
位置：`00-docs/04-项目系统/` 与 `04-projects/`

解决项目怎么选、怎么组织、怎么交付的问题。

## 使用顺序

推荐你按下面顺序推进：

1. 导航线
2. 训练线
3. 环境搭建
4. 执行线
5. 项目线
6. 输出线

## 每周最少要碰到的目录

- `02-daily-reports/`
- `03-weekly-reports/`
- `04-projects/`
- `05-blog-posts/`
- `06-solution-docs/`
- `07-postmortems/`

如果一周结束时这些目录里几乎没有新增内容，说明你大概率只是在“学习”，而不是在“训练”。
