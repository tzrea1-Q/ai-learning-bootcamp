# 多语言文档策略

## 目标

这个仓库希望服务尽量多的人，因此文档体系至少要支持中文和英文。

但双语支持不能靠临时翻译几篇 README，而要有明确的维护边界。

## 当前策略

### 1. 中文仍然是主编辑语言

默认情况下，中文文档是当前仓库的主编辑版本。

### 2. 英文覆盖“核心使用路径”

以下内容必须有英文版本：

- 根入口 README
- 文档导航
- 新手启动路径
- 核心训练主线
- 执行系统核心说明
- 输出系统核心说明
- 项目系统核心说明
- 常用模板

### 3. 非核心扩展文档可分阶段补齐

如果某篇文档只是补充说明，而不是国际用户进入仓库的关键路径，可以先保留中文，再按优先级补英文。

## 目录约定

- 中文主文档继续放在当前目录结构
- 英文文档统一放在 `00-docs/en/`
- 英文模板统一放在 `01-templates/en/`
- 根英文入口使用 `README.en.md`

## 更新规则

每次修改核心中文文档时，都要检查对应英文文档是否需要同步更新。

建议按下面顺序处理：

1. 先改中文主文档
2. 再同步英文镜像文档
3. 最后检查中英文入口链接是否仍然可用

## 最低质量要求

- 英文版本必须完整可读，不能只写标题
- 中英文目录结构尽量保持对应关系
- 关键模板必须让英文使用者可以直接复制使用

## 当前范围

目前已覆盖：

- 根 README 双语入口
- `00-docs` 核心主线英文版
- `01-templates` 核心模板英文版

后续如果英文用户增多，再继续扩展项目级和专题级文档。
