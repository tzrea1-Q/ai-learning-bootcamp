# 项目目录规范

每个正式项目建议按下面结构组织：

```text
project-name/
├─ app/
├─ tests/
├─ docs/
├─ scripts/
├─ .env.example
├─ Dockerfile
├─ docker-compose.yml
├─ requirements.txt
└─ README.md
```

## 目录职责

- `app/`：业务代码
- `tests/`：单元测试与集成测试
- `docs/`：架构、方案、实验记录
- `scripts/`：初始化、导入、调试脚本

## 本地运行产物不属于项目结构

下面这些目录或文件是本地运行产物，不应被当成项目结构的一部分：

- `.env`
- `.venv/`
- `__pycache__/`
- `.pytest_cache/`
- `.mypy_cache/`

要求：

- 这些内容应被 `.gitignore` 忽略
- 项目结构图默认不展示这些本地产物
- 除非你明确要用子模块，否则项目目录里不应出现额外的 `.git/`

## 最低要求

- 必须有 `README.md`
- 必须有 `.env.example`
- 必须有 `tests/`
- 必须能本地运行
- 正式项目建议支持 Docker
