# 项目目录规范

每个正式项目建议按下面结构组织：

```text
project-name/
├─ app/
├─ tests/
├─ docs/
├─ scripts/
├─ .env.example
├─ Dockerfile
├─ docker-compose.yml
├─ requirements.txt
└─ README.md
```

## 目录职责

- `app/`：业务代码
- `tests/`：单元测试与集成测试
- `docs/`：架构、方案、实验记录
- `scripts/`：初始化、导入、调试脚本

## 最低要求

- 必须有 `README.md`
- 必须有 `.env.example`
- 必须有 `tests/`
- 必须能本地运行
- 正式项目建议支持 Docker
