# 项目地图

## 一、为什么要有项目地图

如果你只说“我要做 AI 项目”，大概率会越做越散。

项目地图的作用是把 12 周训练里的项目主线固定下来。

## 二、推荐项目节奏

### 第 1 周

项目类型：最小文本 API

建议目录：`04-projects/week01-text-api/`

### 第 3 周

项目类型：第一个用户可用小产品

可选方向：

- 简历优化助手
- 会议纪要整理器
- 客服回复草稿助手
- 长文总结工具

### 第 5~6 周

项目类型：最小 RAG 系统

### 第 9~10 周

项目类型：多步骤 Workflow / Tool Use 系统

### 第 11~12 周

项目类型：最小 Agent 与作品集整理

## 三、每个项目都必须留下什么

- 代码
- README
- 方案文档
- 测试
- demo 素材
- 复盘或案例

## 四、项目不要做成什么样

- 只有 prompt，没有工程结构
- 只有截图，没有代码
- 只有代码，没有 README
- 只有功能，没有测试和边界说明
