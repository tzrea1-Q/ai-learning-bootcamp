# 贡献指南

感谢你对本项目的关注！欢迎通过 Issue、PR 或 Discussion 参与改进。

## 优先欢迎的反馈

- 文档结构是否清晰
- 新手启动路径是否顺畅
- 示例项目是否真的可复现
- 博客连载和仓库结构是否便于传播

## 提交 Issue

- 用清晰的标题描述问题
- 说明你的环境（操作系统、Python 版本等）
- 如果是 Bug，附上复现步骤

## 提交 PR

1. Fork 本仓库
2. 基于 `main` 分支创建你的功能分支
3. 确保你的改动符合项目现有的目录规范和命名规则
4. 提交清晰的 commit message
5. 发起 Pull Request 并说明改动内容

## 文档维护规则

- 中文是主编辑语言，修改核心中文文档后需检查英文版是否需要同步
- 每条规则只在一个地方维护（Single Source of Truth），其他位置只做链接
- 多语言维护策略详见 [00-docs/00-导航/03-多语言文档策略.md](./00-docs/00-导航/03-多语言文档策略.md)

## 代码风格

- Python 代码遵循 PEP 8
- Markdown 文件使用 UTF-8 编码
- 文件命名使用小写英文 + 连字符（项目代码）或中文（文档）

## 行为准则

请保持友善、包容和建设性的交流态度。
